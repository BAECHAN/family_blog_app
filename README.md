## 팸로그 - 가족블로그

### 1. 목적

가족 구성원들 간의 소통과 추억 공유를 위한 전용 공간을 제공하는 것. 가족끼리만 접근할 수 있는 프라이빗 블로그를 통해 중요한 순간들을 기록하고, 가족들끼리의 유대감을 강화하는 것이 목표입니다.

### 2. 대상 사용자

- **가족 구성원**: 부모님, 형제, 자매, 자녀 등 모든 가족 구성원
- **가까운 친척**: 친척이나 가까운 친척들도 초대할 수 있는 옵션

### 3. 기대 효과

- **가족 유대감 강화**: 중요한 순간들을 기록하고 공유함으로써 가족 구성원들 간의 유대감 강화
- **소통 활성화**: 댓글과 반응 기능을 통해 활발한 소통 유도
- **추억 보존**: 사진 앨범과 포스트를 통해 소중한 추억을 안전하게 보관
- **일정 관리**: 가족 행사나 중요한 일정을 공유하여 더 나은 계획 수립

### 4. 고려사항

- 사용자 경험( UX ) 및 인터페이스( UI ) : 남녀노소 모두 사용하기 쉽고 직관적인 인터페이스 디자인이 필요
- 개인 정보 보호
- 중요한 데이터 손실 방지를 위한 백업 및 복원 기능
- 모바일 개발 관련 ( 일단 웹앱으로만 - 출결 사이트와 같이 )

### 5. 컨텐츠 구성

0. **공통**

   - 카카오톡으로 공유 ( 카카오톡 사용 시 해당 링크 글이 필요할 경우 )
   - 검색
   - 태그
   - 필터
   - 정렬
   - 페이지네이션

1. **로그인**( 담당자 : 지형님)

   - 회원가입 : Oauth 및 관리자권한을 가진 초대코드 전달 ( Oauth든, id/pw 계정 )
     - Oauth : 카카오 혹은 네이버 , 구글 로그인
     - ~~관리자 계정이 수동등록~~
     - 직접 회원가입 : 회원가입 폼 필요
   - 첫 로그인 시 [ 가족 집짓기 / 가족 집들어가기 ] 하나를 선택 할 수 있도록 ( '집들어가기' 컨텐츠 클릭 시 초대코드를 활용 )
   - 선택 후 이어서 자신의 프로필 등록하기 ( 슬라이드 형식의 설문조사 느낌으로 ) - 스킵 고려
   - 인증 및 인가

2. **마이페이지**( 담당자 : 지형님)

   - 회원정보 수정 ( 사진, 생년월일, 좋아하는것, 자주듣는노래, _받고싶은선물_ 등 을 작성하며 이는 '가계도'에서 상세 클릭 시 일부 회원정보를 보이게함)
   - 비밀번호 변경

3. **가계도**

   - 1인칭 기준이 아닌 3인칭으로 ( 아빠 - 엄마 - 아들 - 딸 - 친할아버지 - 외할머니 등... )
   - 가족 사전 : 마이페이지에 작성해둔 내용을 바탕으로 가족 정보( 좋아하는것, 자주듣는노래, _받고싶은선물_ 등 ) 를 볼 수 있음 ( 수정은 마이페이지에서 )
   - 가계도를 그리는 경우 첫째,둘째,셋째.. 아흔아홉번째 이런식으로 순서를 정하기가 어려운데, 일단은 가족 목록만 보여지게
   - 목록에서 가족원 클릭 시 가족사전 페이지로 이동

4. **가족 이벤트 캘린더** ( 담당자 : 희진님 )

   - 월별 캘린더 뷰
   - 가족 구성원들의 생일, 기념일, 휴가 일정 등을 공유할 수 있는 캘린더.
   - 중요한 이벤트에 대한 알림 기능.
   - 메인에서는 '오늘과 내일' 일정만 보여지도록

   - #### 기능
     - 일정 등록
       - 오전 / 오후, 시간대 선택
       - 제목
       - 설명
     - 일정 삭제
     - 일정 수정
     - 일정 보기 ( 캘린더 형태 )

5. **사진 및 동영상 갤러리** ( 담당자 : 찬님 )

   - 가족 모임, 여행 등의 사진과 비디오를 업로드하고 공유할 수 있는 공간.
   - 앨범으로 분류 ( 카드형태, 태그등록(검색을 위한), 사진정보 )

6. **게시판** ( 담당자 : 지형님)

   - 타입 : 공지, 포스트, (투표 : 추후 개발 예정)
   - 공지 타입 : 공지사항이나 토론을 위한 게시판.

     - 배너 형식 - 내부적으로 상세설명
     - 메인에서 보여짐
     - ~~댓글 x, 좋아요 x, 북마크 x~~
     - 최대 2개까지 고정 가능 ( 고정안하면, default: 최신순 )    

   - 포스트 타입 : 각 가족 구성원이 자신의 일상을 공유할 수 있는 블로그 포스트 섹션.

     - 카테고리 폴더로 관리 : 요리 레시피, 여행기, 일상생활, 리뷰 등. ( 카드형식 )

     - #### 기능
        - 카테고리 폴더 추가 / 삭제 / 수정 / 이동 등 관리, ( 카테고리 안에는 [ 공지, 기본게시글 ]를 디폴트로 추가되어있게 )
        - 게시글 등록
        - 게시글 수정
        - 게시글 삭제
        - 게시글 보기
        - 게시글 댓글( 대댓글 )
        - 게시글 이동 ( 드롭다운 형식 )
        - 대댓글
        - 좋아요
        - 북마크

   - 투표 타입 : 가족 여행지 선정, 활동 계획 등을 위한 투표 기능. ( 추후 개발 예정)

7. **쪽지함**

   - 쪽지 전송 (수정 x, 텍스트만 )
   - 쪽지 수신
   - 쪽지 알림

8. **알림**

   - 쪽지 알림
   - 공지 알림
   - 이벤트( 가족생일, 결혼기념일, 어린이날 등 ) 및 일정( 가족여행 등) 알림

9. **관리자**

   - 관리자 UI
   - 가족구성원 관리 ( 초대 링크 또는 관리자 승인 )
   - 특정 게시물에 대해서 지정한 멤버만 볼 수 있도록 지정 ( 추후 개발 예정 )

10. **메인페이지**

- 공지사항 보여주기
- 추천형식으로 "몇년전의 오늘" 또는 "구성원들이 자주 보는" 과 같은 컨텐츠가 상단에 위치
- 캘린더 ( 오늘 , 내일 )

### 6. 프로젝트 기술 스택

#### Vite

차세대 프론트엔드 빌드 도구로, 빠른 개발 서버와 번들링을 제공합니다.

#### FSD (Feature-Sliced Design)

프로젝트 구조를 기능 단위로 나누어, 확장성과 유지보수성을 높이는 프론트엔드 아키텍처입니다.

#### React 18

최신 버전의 React로, 사용자 인터페이스를 구축하기 위한 라이브러리입니다.

#### React Hook Form

React에서 폼을 쉽게 관리하고 검증할 수 있도록 도와주는 라이브러리입니다.

#### TypeScript (TS)

JavaScript의 타입을 확장한 언어로, 코드의 안정성과 가독성을 높여줍니다.

#### React Query

서버 상태를 관리하고, 데이터를 가져오고, 캐시하는 데 도움을 주는 라이브러리입니다.

#### Recoil

React를 위한 상태 관리 라이브러리로, 전역 상태 관리를 쉽게 할 수 있습니다.

#### Supabase

PostgreSQL 기반의 BaaS(Backend-as-a-Service) 플랫폼으로, 실시간 데이터베이스, 인증, 스토리지 기능을 제공합니다.

#### Shadcn

맞춤형 UI 컴포넌트를 쉽게 생성하고 관리할 수 있는 디자인 시스템 라이브러리입니다.
